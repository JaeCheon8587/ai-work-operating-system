# Phase 1/2/3 간략 정리
- Phase 1: PM이 요구사항을 정리하고, 고객 비즈니스 개념·시스템 도메인·스프린트 범위·앱/서비스 담당자를 설계한 뒤 Slack으로 배정 알림을 보내고 CI를 통해 관련 문서를 저장소에 반영한다.
- Phase 2: 개발자/작업자가 Slack 링크로 들어와 자신에게 배정된 앱/서비스 역할을 확인하고, 기능·작업을 등록/수행하며 개발 결과와 변경 이력을 외부 시스템과 연동해 기록한다.
- Phase 3: PM 설계와 개발자들 간에 실행 상태를 연결해 AI가 진행 현황, 의존성, 병목, 우선순위 조정안을 분석하고 관리한다.
